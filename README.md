# BattleShipRGR

Розрахунково-графічна робота з дисципліни "Об’єктно-орієнтоване програмування".
Консольна реалізація гри "Морський бій" на Java з повним дотриманням принципів об'єктно-орієнтованого програмування (ООП).
**Студент** [КМ-41] [Мартинчук Олександра]

## Опис завдання
Розробити гру "Морський бій" (людина проти комп'ютера) з використанням патернів проектування та принципів GRASP.
Гра включає:
- Етап розстановки кораблів (ручний для гравця, автоматичний для ШІ).
- Ігровий цикл (покрокова стрільба).
- Систему сповіщень про події (влучання, промах, потоплення).

## Архітектура та Патерни
У проекті реалізовано наступні архітектурні вимоги:

### 1. Породжувальні шаблони (Creational Patterns) - 3.
* **Factory Method**: Клас `StandardShipFactory` (пакет `model.factory`). Використовується для створення об'єктів кораблів (`Battleship`, `Cruiser` тощо) без прив'язки до їх конкретних класів.
* **Builder**: Клас `GameBuilder` (пакет `model.builder`). Дозволяє покроково конструювати складний об'єкт `GameController`, ініціалізуючи гравців та їх стратегії.
* **Singleton**: Клас `GameSettings` (пакет `util`). Гарантує існування єдиного екземпляра налаштувань гри.

### 2. Поведінкові та Структурні шаблони (Behavioral/Structural) - 4.
* **Strategy**: Інтерфейс `ShootingStrategy` (пакет `model.strategy`). Дозволяє змінювати алгоритм стрільби комп'ютера (наразі реалізовано `RandomShootingStrategy`).
* **Observer**: Інтерфейс `GameObserver` (пакет `observer`). Реалізує реакцію на події гри (влучання/промах) та відділяє логіку виводу (`ConsoleObserver`) від ігрової логіки.
* **Template Method**: Клас `Player` (пакет `model`). Метод `performMove()` задає скелет алгоритму ходу, делегуючи деталі реалізації підкласам (`HumanPlayer`, `ComputerPlayer`).
* **Iterator**: Клас `Board` імплементує `Iterable<Ship>`. Дозволяє перебирати кораблі на полі без розкриття внутрішньої структури списку.

### 3. Принципи GRASP
* **Controller**: Клас `GameController` відповідає за координацію процесу гри, обробку вводу та перемикання ходів.
* **Information Expert**: Клас `Board` володіє всією інформацією про розташування кораблів, тому сам відповідає за перевірку валідності координат та обробку атак.
* **Pure Fabrication**: Клас `InputUtils` (пакет `util`). Штучний клас, створений для винесення логіки парсингу координат, щоб не засмічувати класи логіки (High Cohesion).
* **Low Coupling**: Зв'язність знижена завдяки використанню інтерфейсів (`ShootingStrategy`, `GameObserver`) та ін'єкції залежностей через конструктор контролера.
